# 在 Mac M2 上同时打包 Mac 和 Windows 版本

## 项目技术栈
- **框架**: Tauri 2.x + Vue 3 + TypeScript
- **构建工具**: Vite

## 当前配置
- `src-tauri/tauri.conf.json` 中 `bundle.targets` 已设置为 `"all"`
- 图标已就绪: `icon.icns` (Mac) 和 `icon.ico` (Windows)

## Mac M2 交叉编译配置

### 步骤 1: 安装 Windows 交叉编译工具链

```bash
# 添加 Windows x86_64 目标
rustup target add x86_64-pc-windows-gnu

# 安装 MinGW-w64 工具链（用于 Windows 交叉编译）
brew install mingw-w64



### 步骤 2: 配置 Rust 交叉编译

创建或编辑 `src-tauri/.cargo/config.toml`:

```toml
[target.x86_64-pc-windows-gnu]
linker = "x86_64-w64-mingw32-gcc"
ar = "x86_64-w64-mingw32-ar"
```

### 步骤 3: 修改 tauri.conf.json

**重要修正：** Tauri 2.x 在交叉编译时，`targets` 必须使用 `"all"`（在当前平台打包所有格式）。不能同时指定跨平台的 targets。

修改 `src-tauri/tauri.conf.json`：

```json
"bundle": {
  "active": true,
  "targets": "all",
  ...
}
```

移除 `directory` 配置，因为 Tauri 2.x 不支持在 bundle 下设置输出目录。

可选 targets（单平台）：
| Target | 平台 | 说明 |
|--------|------|------|
| `macos` | macOS | .app 应用包 |
| `dmg` | macOS | .dmg 磁盘镜像 |
| `nsis` | Windows | 安装向导 |
| `msi` | Windows | MSI 安装包 |
| `app` | Linux | AppImage |

**交叉编译说明：** 使用 `--target` 参数时，`tauri.conf.json` 中的 `targets` 应设为 `"all"`，让 CLI 根据目标平台自动选择合适的格式。

### 步骤 4: 执行打包

```bash
# 打包 Mac 版本
npm run tauri build -- --target aarch64-apple-darwin

# 打包 Windows 版本
npm run tauri build -- --target x86_64-pc-windows-gnu
```

### 产出位置
| 平台 | 路径 |
|-----|------|
| Mac | `src-tauri/target/aarch64-apple-darwin/release/bundle/macos/` |
| Windows | `src-tauri/target/x86_64-pc-windows-gnu/release/bundle/windows/` |

### 自定义输出目录

修改 `src-tauri/tauri.conf.json`，在 `bundle` 中添加 `directory` 配置：

```json
"bundle": {
  "active": true,
  "targets": ["macos", "nsis"],
  "directory": "../../builds",
  ...
}
```

修改 `Makefile`，打包后自动重命名并移动到 `./builds`：

```makefile
BUILD_DIR := ./builds
VERSION := $(shell node -p "require('./package.json').version")

build-mac:
	@mkdir -p $(BUILD_DIR)
	@npm run tauri build -- --target aarch64-apple-darwin
	@cp $(DIST_MAC)/*.app $(BUILD_DIR)/install-openclaw-$(VERSION)-macos-arm64.dmg

build-win:
	@mkdir -p $(BUILD_DIR)
	@npm run tauri build -- --target x86_64-pc-windows-gnu
	@cp $(DIST_WIN)/*.exe $(BUILD_DIR)/install-openclaw-$(VERSION)-windows-x86_64.exe
```

最终输出：
```
./builds/
├── install-openclaw-0.1.23-macos-arm64.dmg
├── install-openclaw-0.1.23-windows-x86_64.exe     (直接运行)
└── install-openclaw-0.1.23-windows-x86_64-setup.exe  (安装向导)
```

## 注意事项
1. 首次交叉编译需要下载 rust 依赖，可能需要较长时间
2. 确保 `x86_64-w64-mingw32-gcc` 已正确安装
3. 如果打包失败，检查 `.cargo/config.toml` 配置是否正确

## Makefile 设计

创建 `Makefile` 文件，包含以下目标：

```makefile
.PHONY: help check install-deps build-mac build-win build-all clean

APP_NAME := openclaw-helper
CARGO_DIR := src-tauri
BUILD_DIR := ./builds
VERSION := $(shell node -p "require('./package.json').version")
DIST_MAC := $(CARGO_DIR)/target/aarch64-apple-darwin/release/bundle/dmg
DIST_WIN := $(CARGO_DIR)/target/x86_64-pc-windows-gnu/release/bundle/nsis

help:
	@echo "OpenClaw Helper 打包工具"
	@echo ""
	@echo "可用命令:"
	@echo "  make check         - 检查环境工具是否齐全"
	@echo "  make install-deps  - 安装缺失的依赖（只需执行一次）"
	@echo "  make build-mac     - 打包 Mac 版本 (Apple Silicon)"
	@echo "  make build-win     - 打包 Windows 版本 (x86_64)"
	@echo "  make all           - 同时打包 Mac 和 Windows 版本"
	@echo "  make clean         - 清理构建产物和 builds 目录"

check:
	@echo "检查环境..."
	@which rustc > /dev/null && echo "✓ Rust 已安装" || echo "✗ Rust 未安装"
	@which node > /dev/null && echo "✓ Node.js 已安装" || echo "✗ Node.js 未安装"
	@rustup target list --installed | grep aarch64-apple-darwin > /dev/null && echo "✓ Mac 目标已安装" || echo "✗ Mac 目标未安装 (run: rustup target add aarch64-apple-darwin)"
	@rustup target list --installed | grep x86_64-pc-windows-gnu > /dev/null && echo "✓ Windows 目标已安装" || echo "✗ Windows 目标未安装 (run: rustup target add x86_64-pc-windows-gnu)"
	@which x86_64-w64-mingw32-gcc > /dev/null && echo "✓ MinGW-w64 已安装" || echo "✗ MinGW-w64 未安装 (run: brew install mingw-w64)"


install-deps:
	@echo "安装 Rust Windows 交叉编译目标..."
	@rustup target add x86_64-pc-windows-gnu
	@echo "安装 MinGW-w64..."
	@brew install mingw-w64

	@echo "创建 .cargo/config.toml..."
	@mkdir -p $(CARGO_DIR)/.cargo
	@printf '[target.x86_64-pc-windows-gnu]\nlinker = "x86_64-w64-mingw32-gcc"\nar = "x86_64-w64-mingw32-ar"\n' > $(CARGO_DIR)/.cargo/config.toml
	@echo "完成！"

build-mac:
	@echo "正在打包 Mac 版本..."
	@mkdir -p $(BUILD_DIR)
	@npm run tauri build -- --target aarch64-apple-darwin
	@cp $(DIST_MAC)/*.dmg $(BUILD_DIR)/install-openclaw-$(VERSION)-macos-arm64.dmg
	@echo "Mac 安装包位于: $(BUILD_DIR)/install-openclaw-$(VERSION)-macos-arm64.dmg"

build-win:
	@echo "正在打包 Windows 版本..."
	@mkdir -p $(BUILD_DIR)
	@npm run tauri build -- --target x86_64-pc-windows-gnu || true
	@cp $(CARGO_DIR)/target/x86_64-pc-windows-gnu/release/openclaw-helper.exe $(BUILD_DIR)/install-openclaw-$(VERSION)-windows-x86_64.exe
	@echo "Windows 安装包位于: $(BUILD_DIR)/install-openclaw-$(VERSION)-windows-x86_64.exe"

all: build-mac build-win

clean:
	@echo "清理构建产物..."
	@rm -rf $(CARGO_DIR)/target/release/bundle
	@rm -rf $(BUILD_DIR)
	@echo "完成！"
```

最终输出：
```
./builds/
├── install-openclaw-0.1.23-macos-arm64.dmg
└── install-openclaw-0.1.23-windows-x86_64.exe     (直接运行，无安装向导)
```

**注意**：由于 macOS → Windows 交叉编译时 NSIS 有 std::bad_alloc 限制，Windows 只生成 standalone .exe，不生成安装向导。

## 需要本地安装的工具

| 工具 | 命令 | 用途 | 是否永久 |
|-----|------|------|---------|
| Rust | `brew install rust` | Tauri 运行时 | ✓ 一次安装永久使用 |
| Node.js | `brew install node` | 前端构建 | ✓ 一次安装永久使用 |
| Windows 目标 | `rustup target add x86_64-pc-windows-gnu` | Windows 交叉编译 | ✓ 一次安装永久使用 |
| MinGW-w64 | `brew install mingw-w64` | Windows 交叉编译工具链 | ✓ 一次安装永久使用 |

**总结**: 上述工具都是一次安装永久使用，无需重复安装。

**Windows 输出说明**: Windows 只生成 standalone .exe（免安装绿色版），用户双击即可运行，无需安装。
